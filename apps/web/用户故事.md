# 社区产品 User Story Backlog（V1.0）

## 1. 文档目标

本文档基于《需求文档.md》中的 V1 范围，将产品需求拆分为可执行的 user story backlog，方便用于排期、任务拆分、接口设计和测试用例编写。

## 2. 使用说明

### 2.1 Story 格式

统一采用以下表达方式：

- 作为某类用户
- 我希望完成某个操作
- 从而实现某个业务价值

### 2.2 优先级说明

- P0：首期必须完成，影响核心业务闭环。
- P1：建议首期完成，影响可用性和后台运营。
- P2：可以在首期后半段或下一迭代完成。

### 2.3 状态建议

可在项目管理工具中使用以下状态：

- 待排期
- 开发中
- 联调中
- 测试中
- 已完成

## 3. Epic 列表

- Epic A：账户与身份
- Epic B：钱包与金币
- Epic C：帖子基础能力
- Epic D：资源交易
- Epic E：悬赏问答
- Epic F：评论与互动
- Epic G：消息通知
- Epic H：举报与后台运营

## 4. User Story Backlog

## Epic A：账户与身份

### US-A01 游客浏览公开内容

- 优先级：P0
- 依赖：无
- User Story：作为游客，我希望浏览社区公开内容，从而在注册前了解平台价值并决定是否登录。
- 验收标准：
  - 游客可查看普通帖详情。
  - 游客可查看资源帖公开内容。
  - 游客不可查看资源帖隐藏内容。
  - 游客不可执行评论、点赞、收藏、关注、购买、发帖等登录态操作。
  - 游客点击受限操作时需进入登录流程。

### US-A02 用户注册与登录

- 优先级：P0
- 状态： 完成
- 依赖：无
- User Story：作为新用户，我希望完成注册和登录，从而获得社区的基础使用权限。
- 验收标准：
  - 支持手机号或邮箱注册登录。
  - 注册和登录提交前需完成图形验证码校验，验证码错误时应明确提示并允许重试。
  - 图形验证码需支持刷新，避免被脚本重复利用。
  - 首次注册需补充基础资料。
  - 用户名不可重复，不可使用系统保留词。
  - 登录成功后可进入首页和个人中心。

### US-A03 用户查看和编辑个人资料

- 优先级：P1
- 依赖：US-A02
- User Story：作为登录用户，我希望查看和编辑个人资料，从而建立个人主页并展示身份信息。
- 验收标准：
  - 用户可查看头像、昵称、简介。
  - 用户可查看发帖数、关注数、粉丝数。
  - 用户可编辑头像、昵称、简介。
  - 昵称校验失败时不可保存。

### US-A04 管理员管理用户状态

- 优先级：P1
- 依赖：US-A02
- User Story：作为管理员，我希望管理用户的账号状态，从而维护社区秩序。
- 验收标准：
  - 后台支持查看用户信息和账号状态。
  - 管理员可执行禁言和封禁操作。
  - 禁言用户不可发帖和评论。
  - 封禁用户不可登录。
  - 所有后台操作需记录时间、操作者和原因。

## Epic B：钱包与金币

### US-B01 用户获得初始金币钱包

- 优先级：P0
- 依赖：US-A02
- User Story：作为新注册用户，我希望拥有金币钱包，从而能够参与资源购买和悬赏问答。
- 验收标准：
  - 注册成功后自动创建钱包。
  - 钱包至少包含可用金币和冻结金币两个字段。
  - 系统按运营规则发放初始金币。
  - 金币余额不可为负数。

### US-B02 用户查看金币流水

- 优先级：P0
- 依赖：US-B01
- User Story：作为用户，我希望查看金币流水，从而知道每一笔资产变化的原因。
- 验收标准：
  - 个人中心支持查看金币流水列表。
  - 流水展示变动时间、金额、方向、业务类型、备注。
  - 至少覆盖注册赠送、后台加币、购买扣减、资源收入、悬赏冻结、悬赏发放、悬赏退回。

### US-B03 管理员后台加币与扣币

- 优先级：P1
- 依赖：US-B01
- User Story：作为管理员，我希望对用户执行加币或扣币，从而支持早期运营和异常处理。
- 验收标准：
  - 后台支持按用户执行加币和扣币操作。
  - 操作时必须填写原因。
  - 操作成功后同步生成流水记录。
  - 扣币后余额不得为负数。

### US-B04 资产操作一致性保障

- 优先级：P0
- 依赖：US-B01
- User Story：作为平台，我希望金币相关操作保持一致性，从而避免重复扣费、重复发放或资产异常。
- 验收标准：
  - 购买、冻结、退回、发放必须同时更新账户和流水。
  - 同一笔业务重复提交不得重复扣费或重复发奖。
  - 失败操作不得留下半成功状态。

## Epic C：帖子基础能力

### US-C01 用户发布不同类型帖子

- 优先级：P0
- 依赖：US-A02
- User Story：作为登录用户，我希望发布普通帖、资源帖和悬赏帖，从而完成不同内容表达和交易诉求。
- 验收标准：
  - 发帖时必须选择帖子类型。
  - 不同类型展示不同字段。
  - 发帖成功后帖子默认为已发布状态。
  - 未登录用户不可发帖。

### US-C02 用户浏览帖子列表和详情

- 优先级：P0
- 依赖：US-C01
- User Story：作为用户，我希望浏览帖子列表和详情，从而找到感兴趣的内容。
- 验收标准：
  - 首页列表默认按最新发布排序。
  - 列表可区分普通帖、资源帖、悬赏帖。
  - 详情页展示标题、作者、发布时间、正文、状态和互动数据。

### US-C03 作者编辑和删除自己的帖子

- 优先级：P1
- 依赖：US-C01
- User Story：作为发帖人，我希望编辑、删除和关闭自己的帖子，从而管理内容生命周期。
- 验收标准：
  - 作者可编辑未下架帖子。
  - 作者可关闭帖子。
  - 删除采用逻辑删除。
  - 已发生的购买记录和赏金记录不因编辑消失。

### US-C04 管理员下架帖子

- 优先级：P1
- 依赖：US-C02
- User Story：作为管理员，我希望下架违规帖子，从而阻断不合规内容继续传播。
- 验收标准：
  - 后台支持按作者、状态、类型筛选帖子。
  - 管理员可执行下架操作。
  - 下架后帖子前台不可见。
  - 下架原因需保留记录。

## Epic D：资源交易

### US-D01 发布资源帖

- 优先级：P0
- 依赖：US-C01
- User Story：作为内容发布者，我希望发布资源帖并设置金币售价，从而让资源内容产生收益。
- 验收标准：
  - 资源帖需填写标题、公开内容、隐藏内容、售价。
  - 售价必须是大于 0 的整数。
  - 发布后资源帖默认进入售卖中状态。

### US-D02 查看资源帖公开和隐藏内容

- 优先级：P0
- 依赖：US-D01
- User Story：作为浏览用户，我希望先查看资源帖公开内容，再决定是否购买，从而降低决策成本。
- 验收标准：
  - 游客和未购买用户仅可查看公开内容。
  - 已购买用户和作者可查看隐藏内容。
  - 隐藏内容区域需展示明确的状态提示和购买按钮。
  - 已购买用户按钮状态展示为已解锁。

### US-D03 用户购买资源帖

- 优先级：P0
- 依赖：US-B04, US-D02
- User Story：作为用户，我希望使用金币购买资源帖，从而获得隐藏内容访问权限。
- 验收标准：
  - 购买前校验登录状态、余额、是否已购买、是否为本人帖子。
  - 余额不足时购买失败并提示原因。
  - 重复点击购买不得重复扣费。
  - 购买成功后立即可查看隐藏内容。
  - 同步生成交易记录、买卖双方流水和消息通知。

### US-D04 买家查看购买记录

- 优先级：P1
- 依赖：US-D03
- User Story：作为买家，我希望查看资源购买记录，从而管理我的已购内容。
- 验收标准：
  - 个人中心展示已购买资源列表。
  - 列表展示资源标题、购买时间、支付金币、状态。
  - 用户可从列表回到详情页查看已解锁内容。

### US-D05 卖家查看资源销售记录

- 优先级：P1
- 依赖：US-D03
- User Story：作为卖家，我希望查看资源销售记录和累计收益，从而了解内容变现效果。
- 验收标准：
  - 卖家可查看被购买记录。
  - 卖家可查看购买人数和累计收入。
  - 后台可按用户、帖子、时间查询相关交易。

### US-D06 买家发起资源申诉

- 优先级：P1
- 依赖：US-D03
- User Story：作为买家，我希望在资源失效或内容不符时发起申诉，从而获得平台处理。
- 验收标准：
  - 已购买资源支持提交申诉。
  - 申诉需填写原因和补充说明。
  - 申诉进入后台待处理列表。
  - 管理员处理后同步通知用户。

### US-D07 管理员处理资源退款申诉

- 优先级：P1
- 依赖：US-D06, US-B04
- User Story：作为管理员，我希望处理资源申诉并执行退款或补偿，从而保障交易秩序。
- 验收标准：
  - 管理员可对申诉标记有效或无效。
  - 支持执行部分退币或全额退币。
  - 退款成功后同步更新账户、流水、交易状态和消息通知。

## Epic E：悬赏问答

### US-E01 发布悬赏帖

- 优先级：P0
- 依赖：US-B04, US-C01
- User Story：作为提问用户，我希望发布悬赏帖并冻结金币，从而吸引其他用户回答问题。
- 验收标准：
  - 发布时需填写标题、问题描述、悬赏金额和有效期。
  - 余额不足时发布失败。
  - 发布成功后冻结相应金币。
  - 帖子状态变为进行中。

### US-E02 用户提交候选答案

- 优先级：P0
- 依赖：US-E01
- User Story：作为回答用户，我希望在悬赏帖下提交候选答案，从而参与竞争赏金。
- 验收标准：
  - 一级评论视为候选答案。
  - 二级评论仅用于追问和讨论。
  - 发帖人不可把自己的评论作为候选答案。
  - 登录用户才可回答。

### US-E03 发帖人采纳答案

- 优先级：P0
- 依赖：US-E02, US-B04
- User Story：作为提问用户，我希望采纳一个最佳答案，从而把冻结赏金发给答主。
- 验收标准：
  - 仅发帖人可执行采纳。
  - 每篇悬赏帖只能采纳一次。
  - 采纳后冻结金币全部转给被采纳答主。
  - 采纳后帖子状态变为已解决。
  - 同步生成流水和交易通知。

### US-E04 悬赏帖超时自动退回

- 优先级：P0
- 依赖：US-E01, US-B04
- User Story：作为提问用户，我希望悬赏超时后自动退回冻结金币，从而避免资产长期占用。
- 验收标准：
  - 超过有效期未采纳时自动退回冻结金币。
  - 帖子状态改为已结束未采纳。
  - 已结束未采纳的帖子不可再次采纳。
  - 同步生成退回流水和消息通知。

### US-E05 管理员处理悬赏异常

- 优先级：P1
- 依赖：US-E03, US-E04
- User Story：作为管理员，我希望处理恶意悬赏、灌水回答和争议采纳，从而维护问答秩序。
- 验收标准：
  - 后台可查看悬赏状态和回答记录。
  - 管理员可关闭帖子和删除违规回答。
  - 如涉及资产调整，需保留操作记录和原因。

## Epic F：评论与互动

### US-F01 用户发表评论

- 优先级：P0
- 依赖：US-A02, US-C02
- User Story：作为登录用户，我希望在帖子下发表评论，从而参与社区讨论。
- 验收标准：
  - 仅登录用户可评论。
  - 禁言用户不可评论。
  - 评论成功后帖子评论数更新。

### US-F02 用户回复一级评论

- 优先级：P1
- 依赖：US-F01
- User Story：作为用户，我希望回复一级评论，从而在具体楼层内展开交流。
- 验收标准：
  - 二级评论只能回复一级评论。
  - 二级评论需显示回复对象。
  - 不支持三级嵌套。

### US-F03 用户删除自己的评论

- 优先级：P1
- 依赖：US-F01
- User Story：作为评论作者，我希望删除自己的评论，从而修正不当内容。
- 验收标准：
  - 用户只能删除自己的评论。
  - 删除后前台显示“该评论已删除”。
  - 删除采用逻辑删除。

### US-F04 用户点赞帖子和评论

- 优先级：P0
- 依赖：US-C02, US-F01
- User Story：作为用户，我希望点赞帖子和评论，从而表达认可。
- 验收标准：
  - 同一用户对同一对象仅可点赞一次。
  - 再次点击可取消点赞。
  - 点赞数需正确更新。

### US-F05 用户收藏帖子

- 优先级：P1
- 依赖：US-C02
- User Story：作为用户，我希望收藏帖子，从而方便后续回看。
- 验收标准：
  - 仅帖子支持收藏。
  - 收藏后进入个人收藏列表。
  - 用户可取消收藏。

### US-F06 用户关注其他用户

- 优先级：P1
- 依赖：US-A02
- User Story：作为用户，我希望关注其他用户，从而持续获取对方的内容动态。
- 验收标准：
  - 不可关注自己。
  - 关注和取消关注均需成功更新关注数、粉丝数。
  - 用户主页展示关注关系数据。

## Epic G：消息通知

### US-G01 接收互动类通知

- 优先级：P1
- 依赖：US-F04, US-F05, US-F06
- User Story：作为用户，我希望收到点赞、收藏和关注通知，从而知道内容反馈和关系变化。
- 验收标准：
  - 点赞、收藏、关注均可触发通知。
  - 消息展示触发人、时间和关联内容。
  - 点击消息可跳转对应内容。

### US-G02 接收交流类通知

- 优先级：P1
- 依赖：US-F01, US-F02
- User Story：作为用户，我希望收到评论、回复和 @ 通知，从而及时参与讨论。
- 验收标准：
  - 评论、回复、@ 分别触发通知。
  - 消息按时间倒序展示。
  - 未读数需正确累计。

### US-G03 接收交易类通知

- 优先级：P0
- 依赖：US-D03, US-D07, US-E03, US-E04
- User Story：作为用户，我希望收到资源购买、退款、悬赏采纳和悬赏退回通知，从而及时感知资产变化。
- 验收标准：
  - 资源被购买、资源退款、悬赏采纳、悬赏退回均生成通知。
  - 通知内容可区分交易类型。
  - 通知可跳转到对应内容或记录详情。

### US-G04 消息已读和未读管理

- 优先级：P1
- 依赖：US-G01, US-G02, US-G03
- User Story：作为用户，我希望查看全局未读数并管理已读状态，从而高效处理消息。
- 验收标准：
  - 顶部支持展示全局未读数。
  - 支持单条已读。
  - 支持全部已读。
  - 已读后未读数同步减少。

### US-G05 高频互动通知聚合

- 优先级：P2
- 依赖：US-G01
- User Story：作为用户，我希望高频点赞类消息被聚合展示，从而避免消息过载。
- 验收标准：
  - 同一帖子短时间内多个点赞可聚合。
  - 评论、交易、处罚类通知不聚合。
  - 聚合消息文案可展示多用户行为。

## Epic H：举报与后台运营

### US-H01 用户举报帖子和评论

- 优先级：P1
- 依赖：US-C02, US-F01
- User Story：作为登录用户，我希望举报帖子和评论，从而反馈违规内容。
- 验收标准：
  - 帖子和评论均支持举报。
  - 举报时必须选择举报原因。
  - 举报提交后进入后台待处理列表。

### US-H02 管理员处理举报

- 优先级：P1
- 依赖：US-H01
- User Story：作为管理员，我希望处理举报记录，从而完成违规内容治理。
- 验收标准：
  - 后台可查看举报详情和历史记录。
  - 管理员可标记有效、无效、已处罚。
  - 可联动下架帖子、删除评论、处罚用户。

### US-H03 管理员查看交易和资产审计记录
