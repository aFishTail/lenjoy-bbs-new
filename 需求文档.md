# 社区产品需求文档 PRD（V1.0）

## 1. 文档说明

### 1.1 文档目标

本文档用于明确社区产品 V1 的业务目标、功能范围、角色权限、业务流程、状态规则、异常处理和后台管理要求，供产品、设计、开发、测试共同评审与落地。

### 1.2 产品定位

本产品是一个围绕内容分享、知识交流和资源变现的互动社区。平台通过金币体系连接内容消费与内容供给，支持普通讨论、付费资源分享、悬赏问答三类核心场景。

### 1.3 版本边界

- V1 聚焦站内金币闭环，不接第三方支付，不支持提现。
- V1 支持后台人工加金币，作为早期运营与资产发放手段。
- 第三方支付、自动充值、分账、提现属于后续版本能力，本期只预留扩展位，不进入开发范围。

### 1.4 目标用户

- 有资料分享和内容变现诉求的内容发布者。
- 有资源获取和问题求助诉求的普通用户。
- 需要管理社区秩序、处理举报与运营资产的后台人员。

## 2. 产品目标

### 2.1 业务目标

1. 搭建一个可承载内容讨论、资源售卖、悬赏问答的社区基础平台。
2. 通过金币体系建立用户付费和内容激励闭环。
3. 通过互动、通知、关注关系提升用户活跃与留存。
4. 通过后台运营工具支撑早期内容治理和人工运营。

### 2.2 V1 成功标准

1. 用户可完成注册、登录、浏览、发帖、评论、点赞、收藏、关注。
2. 用户可使用金币购买资源帖，可查看自己的交易记录。
3. 用户可发布悬赏帖、回答问题、采纳答案、退回超时赏金。
4. 管理员可处理举报、上下架帖子、后台加币、查看流水与交易记录。

## 3. 角色定义与权限矩阵

### 3.1 角色定义

- 游客：未登录用户，仅可浏览公开内容。
- 普通用户：已注册登录用户，可进行基础互动、发帖、购买、回答。
- 发帖人：针对某篇帖子拥有编辑、删除、关闭、采纳等特定权限的用户。
- 答主：在悬赏帖中发表评论并参与回答的用户。
- 管理员：拥有后台审核、内容管理、用户管理、资产运营权限。

| 浏览资源帖公开内容 | 支持   | 支持                 | 支持                 | 支持         |
| 查看资源帖隐藏内容 | 不支持 | 购买后支持           | 支持                 | 支持         |

## 4. 核心业务流程

### 4.1 用户主流程

1. 用户注册并登录。
2. 用户完善个人资料，获得初始金币。
3. 用户浏览社区内容，可进行评论、点赞、收藏、关注。
4. 用户可发布普通帖、资源帖或悬赏帖。
5. 用户可购买资源帖或回答悬赏帖。
6. 用户在消息中心接收互动、交易和交流通知。

### 4.2 资源交易流程

1. 发帖人发布资源帖，填写公开内容、隐藏内容和售价。
2. 浏览用户查看公开内容后，点击购买。
3. 系统校验用户登录状态、金币余额、是否已购买、是否为本人帖子。
4. 校验通过后扣减买家金币，并增加卖家可用金币。
5. 系统生成交易记录、金币流水、通知消息。
6. 买家获得资源隐藏内容的永久查看权限。

### 4.3 悬赏问答流程

1. 发帖人发布悬赏帖并设置赏金金额。
2. 系统冻结对应金币，帖子进入进行中状态。
3. 其他用户在评论区提交一级评论作为候选答案。
4. 发帖人在有效期内选择采纳某条一级评论。
5. 采纳后冻结金币转入答主账户，帖子进入已解决状态。
6. 若超过有效期未采纳，系统自动退回冻结金币，帖子进入已结束未采纳状态。

### 5.1 V1 必做

- 注册登录
- 个人主页
- 金币钱包与流水
- 普通帖、资源帖、悬赏帖

- 点赞、收藏、关注

### 5.2 V1.1 候选

- 首页推荐策略升级
- 新手任务与签到送金币

## 6. 用户与账户体系

### 6.1 注册与登录

- 支持手机号或邮箱注册，具体登录方式可由开发阶段结合现有账号体系确定。
- 用户首次注册需完成基础信息填写。
- 用户名默认唯一，不允许与系统保留词重复。
- 未登录用户执行需要授权的操作时，统一拉起登录。

### 6.2 用户资料

- 基础资料字段：头像、昵称、个人简介。
- 用户主页展示：发帖数、获赞数、关注数、粉丝数、金币余额入口。
- 用户可编辑自己的资料，管理员可在后台查看与处理异常账号。

### 6.3 账号状态

- 正常：可正常使用全部前台功能。
- 禁言：不可发帖、评论，但可登录浏览。
- 封禁：不可登录，不可使用前台功能。
- 注销预留：V1 不开放自主注销，仅预留后台处理能力。

## 7. 金币体系与资产规则

### 7.1 金币定义

金币是站内虚拟资产，用于购买资源帖、发布悬赏帖和后续平台激励场景。

### 7.2 金币来源

- 新用户注册赠送初始金币。
- 管理员后台加金币。
- 资源帖被购买后获得金币收入。
- 悬赏帖答案被采纳后获得金币奖励。
- 后续版本可扩展第三方充值入口。

### 7.3 金币去向

- 购买资源帖时扣减。
- 发布悬赏帖时冻结。
- 后台处罚时可人工扣减。

### 7.4 钱包字段

- 可用金币：用户当前可直接消费的金币。
- 冻结金币：因悬赏等场景临时冻结的金币。
- 累计收入金币：用于用户侧展示历史获得金币总量，可选。

### 7.5 流水记录要求

每次金币变动必须生成流水，至少包含以下字段：

- 流水编号
- 用户 ID
- 变动类型
- 变动方向（收入/支出/冻结/解冻）
- 变动金额
- 变动前余额
- 变动后余额
- 关联业务类型
- 关联业务 ID
- 备注
- 创建时间

### 7.6 金币规则

1. 金币不可为负数。
2. 同一资源帖同一用户只扣费一次，购买记录永久有效。
3. 用户不能购买自己的资源帖。
4. 发布悬赏帖时需校验可用金币是否足够。
5. 悬赏未结束前，对应赏金不可再次使用。
6. 任何资产操作失败时，不得只写入部分记录，必须保证账户、流水、交易状态一致。

### 7.7 未来支付扩展位

- 预留充值订单概念。
- 预留充值来源字段，如后台发放、活动赠送、第三方支付。
- 当前版本不展示充值入口，不接支付回调。

## 8. 帖子系统总规则

### 8.1 帖子通用字段

- 帖子类型：普通帖、资源帖、悬赏帖。
- 标题。
- 正文内容。
- 封面图，可选。
- 作者。
- 发布时间。
- 状态。
- 点赞数、评论数、收藏数、浏览数。

### 8.2 帖子通用状态

- 草稿：仅作者可见。
- 已发布：前台可正常浏览。
- 已关闭：禁止新增评论和新交易，但历史内容仍可查看。
- 已下架：前台不可见，管理员可后台查看。
- 已删除：逻辑删除，仅管理员可追溯。

### 8.3 发帖规则

1. 仅登录用户可发帖。
2. 发帖时必须选择帖子类型。
3. 不同帖子类型展示不同表单项。
4. 发帖成功默认进入已发布状态。
5. 作者可编辑未下架帖子，编辑后不影响已发生的交易记录。

## 9. 普通讨论帖

### 9.1 定义

用于普通内容讨论、经验分享、资讯发布。

### 9.2 发布字段

- 标题，必填。
- 正文，必填。
- 图片，可选。

### 9.3 展示规则

- 所有人可查看。
- 登录用户可评论、点赞、收藏。
- 作者可关闭评论。

### 9.4 管理规则

- 作者可编辑、删除自己的帖子。
- 管理员可下架违规帖子。

## 10. 付费资源帖

### 10.1 定义

资源帖用于分享有价值内容，并对隐藏内容设置金币解锁门槛。

### 10.2 发布字段

- 标题，必填。
- 公开内容，必填，用于介绍资源价值和使用说明。
- 隐藏内容，必填，可包含下载链接、提取码、使用方式、资源说明等。
- 售价，必填，必须为大于 0 的整数。
- 封面图，可选。

### 10.3 前台展示规则

- 游客和未购买用户仅可查看公开内容。
- 已购买用户和作者可查看隐藏内容。
- 隐藏内容区域需有明确的购买状态提示。
- 若用户已购买，按钮显示为已解锁。

### 10.4 购买规则

1. 用户必须登录后才能购买。
2. 用户不能购买自己的资源帖。
3. 用户余额不足时，提示金币不足。
4. 用户重复点击购买时，不得重复扣费。
5. 购买成功后立即开通查看权限。

### 10.5 交易记录

- 买家侧可查看购买时间、资源标题、支付金币、当前状态。
- 卖家侧可查看被购买记录、购买人数、累计收入。
- 管理员可按帖子、用户、时间查询交易。

### 10.6 退款与申诉规则

- V1 默认不支持用户自主退款。
- 用户若认为资源失效、内容不符或无法使用，可发起申诉。
- 管理员人工判定后，可执行补偿、部分退币或全额退币。
- 退款完成后需同步生成退款流水与通知。

### 10.7 资源帖状态

- 售卖中：可正常购买。
- 已关闭：前台保留展示，但不可新增购买。
- 已下架：前台不可见，已购买用户是否保留访问权由管理员配置，V1 默认保留已购用户查看权。

## 11. 悬赏问答帖

### 11.1 定义

悬赏帖用于用户提出问题并以金币激励其他用户回答。

### 11.2 发布字段

- 标题，必填。
- 问题描述，必填。
- 悬赏金额，必填，必须为大于 0 的整数。
- 有效期，必填，V1 默认支持 1 天、3 天、7 天。
- 补充图片，可选。

### 11.3 发布规则

1. 发布前需校验用户可用金币是否充足。
2. 发布成功后，对应金币进入冻结状态。
3. 帖子创建成功即进入进行中状态。

### 11.4 回答规则

1. 仅一级评论视为候选答案。
2. 二级评论仅用于追问、讨论，不参与采纳。
3. 同一用户可提交多个一级评论，V1 允许，但最终只可有一条被采纳。
4. 发帖人不可回复自己的悬赏帖作为候选答案。

### 11.5 采纳规则

1. 仅发帖人可执行采纳。
2. 每篇悬赏帖只能采纳一次。
3. 采纳后冻结金币全部转给被采纳答主。
4. 采纳后帖子状态变为已解决，不可再次采纳。
5. 采纳后普通评论功能保留，便于补充交流。

### 11.6 超时规则

1. 超过有效期未采纳，系统自动将冻结金币全额退回发帖人。
2. 超时后帖子状态变为已结束未采纳。
3. 已结束未采纳的帖子不允许再执行采纳。

### 11.7 后台干预规则

- 如发生恶意悬赏、灌水回答、争议采纳，管理员可介入。
- 管理员可关闭帖子、删除违规回答、按运营规则人工调整金币。

## 12. 评论与互动系统

### 12.1 评论结构

- 一级评论：直接回复帖子。
- 二级评论：回复一级评论。
- V1 采用两级评论展平结构，不支持三级及以上层级。

### 12.2 评论规则

1. 仅登录用户可评论。
2. 被禁言用户不可评论。
3. 一级评论可被回复。
4. 二级评论展示在对应一级评论下方，并显示回复对象。
5. 评论支持删除自己的内容，删除后显示“该评论已删除”。

### 12.3 点赞规则

- 支持对帖子点赞。
- 支持对评论点赞。
- 同一用户对同一对象仅可点赞一次。
- 再次点击可取消点赞。

### 12.4 收藏规则

- 仅支持收藏帖子。
- 收藏后进入个人收藏列表。
- 用户可取消收藏。

### 12.5 关注规则

- 用户可关注其他用户。
- 不可关注自己。
- 关注后可在个人主页看到关注关系。

## 13. 消息通知中心

### 13.1 消息分类

- 互动类：点赞、收藏、关注。
- 交流类：评论、回复、@。
- 交易类：资源被购买、资源退款、悬赏被采纳、悬赏退回。
- 系统类：后台加币、处罚通知、举报处理结果。

### 13.2 消息展示规则

- 消息中心按时间倒序展示。
- 每条消息需展示消息类型、触发人、关联内容、触发时间。
- 点击消息可跳转到对应内容详情。

### 13.3 聚合规则

- 同一帖子在短时间内收到多个点赞时可聚合展示。
- 聚合文案示例：A、B 等 10 人赞了你的帖子。
- 评论、交易、处罚类消息不聚合，避免影响处理。

### 13.4 未读规则

- 顶部展示全局未读数。
- 进入消息中心后，可按分类查看消息。
- 用户可单条已读或一键全部已读。

## 14. 搜索、筛选与列表

### 14.1 首页列表

- 默认按最新发布排序。
- 支持按帖子类型切换。
- 支持基础热门排序扩展位，V1 可先不做复杂推荐。

### 14.2 搜索范围

- 支持按标题和正文关键字搜索帖子。
- 支持按帖子类型筛选结果。

### 14.3 个人中心列表

- 我的帖子。
- 我的购买。
- 我的悬赏。
- 我的收藏。
- 我的金币流水。

## 15. 举报与内容治理

### 15.1 举报入口

- 帖子支持举报。
- 评论支持举报。
- 举报入口仅登录用户可见。

### 15.2 举报原因

- 广告营销。
- 违规内容。
- 资源无效。
- 内容抄袭。
- 恶意辱骂。
- 其他。

### 15.3 处理机制

- 举报提交后进入后台待处理列表。
- 管理员可标记为有效、无效、已处罚。
- 若涉及资源退款或处罚扣币，需记录处理备注。

## 16. 后台运营系统

### 16.1 用户管理

- 查看用户基础信息。
- 查看金币余额、冻结金币、流水记录。
- 执行加币、扣币、禁言、封禁操作。

### 16.2 帖子管理

- 按帖子类型、状态、作者、时间筛选。
- 支持下架、删除、关闭评论、查看举报记录。

### 16.3 交易管理

- 查看资源交易记录。
- 查看悬赏冻结、发放、退回记录。
- 支持按用户、帖子、时间筛选。

### 16.4 举报与申诉管理

- 查看举报列表。
- 查看资源申诉。
- 处理退款、处罚、恢复上架等动作。

### 16.5 运营看板

V1 至少提供以下基础指标：

- 新增用户数。
- 发帖数。
- 资源购买单数。
- 悬赏发布数与采纳率。
- 金币发放总量与消耗总量。

## 17. 异常场景与边界规则

### 17.1 资源购买异常

- 余额不足：购买失败，不产生交易记录。
- 重复购买：直接返回已购买状态，不重复扣费。
- 帖子已关闭或下架：不可购买。
- 作者本人购买：前端不展示购买按钮，后端也需拦截。

### 17.2 悬赏异常

- 余额不足：发布失败。
- 帖子超时：自动退回赏金。
- 目标答案已删除：不可采纳。
- 帖子已采纳：不可再次采纳。

### 17.3 账号异常

- 禁言用户不可发帖、评论。
- 封禁用户不可登录。
- 用户被封禁后，已购资源记录仍保留，具体访问策略由管理员控制，V1 默认保留历史购买记录。

## 18. 数据与埋点建议

### 18.1 关键指标

- 日活用户数。
- 发帖转化率。
- 资源帖购买转化率。
- 悬赏帖采纳率。
- 评论参与率。
- 消息打开率。

### 18.2 建议埋点

- 发帖提交。
- 资源购买点击。
- 资源购买成功。
- 悬赏发布成功。
- 采纳成功。
- 举报提交。
- 消息点击。

## 19. 非功能要求

### 19.1 性能要求

- 列表页、详情页、消息页需支持分页。
- 金币扣减、冻结、退回必须保证数据一致性。
- 高并发场景下不得出现重复扣费或多次发奖。

### 19.2 安全要求

- 所有需要登录的动作必须校验身份。
- 金币相关接口必须做重复提交防护。
- 帖子与评论内容需具备基础敏感词和违规内容审核能力。

### 19.3 终端要求

- V1 前台需兼容 PC 和移动端浏览。
- 后台优先面向 PC 使用。

## 20. 评审待确认项

1. 新用户初始金币发放数量。
2. 资源帖售价上下限。
3. 悬赏金额上下限。
4. 悬赏有效期是否仅支持固定选项。
5. 是否允许作者关闭资源帖后继续让已购用户访问。
6. 是否需要对评论区增加排序方式。
7. 是否需要增加标签和分类体系作为 V1 范围。

## 21. 交付建议

开发落地时可按以下顺序拆分：

1. 账户与金币体系。
2. 帖子基础能力。
3. 资源帖交易闭环。
4. 悬赏问答闭环。
5. 评论互动与消息中心。
6. 举报审核与后台运营。

以上文档作为 V1 产品需求基线，后续新增需求应以版本迭代形式补充。
